Tableau Environment

Welcome to Tableau! This is a powerful and widely used software for creating data visualizations which is particularly useful for “non-programmers”. It has a point-and-click menu system and intuitive drag-and-drop plot creation tool that allows you to quickly make detailed, appealing graphs. Tableau also has powerful data joining tools (allowing you to link different data files), hosting services, and interactive project dashboards that make for great presentation and consulting tools. If your objective is to tell stories with quantitative data and make it widely available to diverse audiences, Tableau is definitely for you!

Importing Data

Tableau can handle a wide variety of data types:

  • Microsoft Excel (.xls, .xlsx)
  • Text (.csv, .txt, .tab)
  • Microsoft Access (.mdb)
  • Spatial (.shp, .kml)
  • JSON (.json)
  • Many more!

Tableau also interfaces with server databases, including the large and powerful Tableau Online servers (not included with Student edition). This trial will focus on using our own data (from open source providers) centered on New York City.

To import data, use the menu on the left side of the start page to pick the type of data you want to use. We will start with a .csv file of New York demographics from the American Community Survey (ACS). The ACS is an ongoing effort by the US Census Bureau to provide updated information on the people of the United States organized by various levels of geography. The Census has its own specific geographic categories, with diagrams available here, that starts at the national level and gets more specific as it disaggregates: regions, divisions, states, counties, census tracts, block groups, and finally census blocks. Thanks to the statistical sampling methods used by the Census bureau for updating its ACS data each year, the accuracy of estimate data decreases as geographic specificity increases. In other words, it’s hard to find accurate data at the census block level. Census tracts are usually the standard for cross-state comparisons. We will use census block groups as a compromise between data accuracy and geographic specificity.

Select the “Text file” option under “Connect - To a File”. You will see the above window appear, which you can use to browse to the “New_York_City” folder and the delimited text file “NYC_ACS_Data_Perc.csv”. Hit open to bring the data into Tableau.

You will see the above screen automatically when you import the .csv file. Tableau has automatically created a workbook of your current session, which you can save using the “File - Save as…” dropdown menu. Like Stata and many other programs, this file is only a collection of Tableau instructions and doesn’t contain the underlying data. For others to use your file you will need to send the source data!

Examine the ACS data we just pulled in through Tableau’s “Data Source” tab. On the top half of the screen you see your data environment. Currently it only has one object--your ACS data file. You can connect many different files into a single data environment and visualize the relationships here.

Below the data environment is a sample view of the .csv file, displayed like an Excel table with rows as observations and columns as variables. Notice that each column has the variable name in bold, the data source in light gray, and the variable data type in blue. Variables can be coded as numeric, strings (character text), dates, booleans (binary logic of 0’s and 1’s), and geography. Geography only accepts certain location types, such as city, country, state, county, etc. Other geographies, like our census block groups, are stored as either character strings (like the “Geography” variable, confusingly) or numeric ID’s (“Id” and “Id2”). Descriptive variables (like “Pop.Estimate”) are also available if you scroll over to the right. The formats for these variables were set by the “NYC_ACS_Data_Perc.csvt” file, a .csv support file that helps programs like Tableau and QGIS know what format to put variables in (since Excel doesn’t keep formats in .csv files); for more read here.

You can sort and otherwise explore your data in Tableau similarly to Excel. One key difference is the option bubbles at the top right, “Live” and “Extract”. Tableau has a powerful option to maintain your work in connection to existing files, updating its analyses and visualizations automatically if you edit the data in another program like Excel. As long as you save the file with the same name, Tableau will automatically update as long as you set it to “Live”. This takes up more computing power, naturally, so if you just want to work with a temporary copy of the data you can select “Extract”.

Creating Plot Visualizations

To get started making visualizations in Tableau, click on “Sheet 1” in the tabs on the bottom left. You’ll see an empty graph window with a list of variables on the left under the “Data” window. Tableau organizes your variables automatically into three categories: Dimensions, Measures, and Attributes. Dimensions and Attributes are rather similar--both describe values for individual observations in the data. Measures are used to aggregate data across observations, to represent the average value for an entire county from census block groups, for example. For more on the differences between Measures and Dimensions, read on here (also includes Continuous vs. Discrete and Aggregate vs. Non-Aggregate). For a brief discussion on the difference between Dimensions and Attributes, read this forum post.

To start creating a plot, simply click and drag a variable into your visualization window, either in the graph field itself or up in the “Rows” and “Columns”. To follow along with the screenshots, drag “HH Inc 150k to 200k Perc” from the Measures list into the Rows space. You’ll notice that it only displays one value in a bar--this is because the variable is formatted as a Measure (Sum). Right click on it in Rows to change to a Dimension, as below.

Tableau is limited in displaying single variables. Go ahead and drag in a second variable to plot against income--we used “Edu Master or Prof Degr Perc” in the Columns space. Change it to a Dimension as well.

You should now see a bunch of observations organized in a scatter plot with the percentage of the population with a master or professional degree on the x-axis and percentage of the population with incomes between $150k - $200k on the y-axis. Tableau tries to format your plot in the appropriate format based on your data types automatically, though you can adjust this quickly via the “Show Me” button in the top right.

Our scatter plot is formatted by default as empty blue circles. You can see many observations overlap towards the bottom of the graph. We’ll filter out the majority of observations to make the plot more meaningful, but first let’s play with the “Marks” options. Tableau allows you to change the color, shape, size, label, and details of observations by variables through the same drag-and-drop process. Let’s drag “Pop.Estimate” over to the Size card.

You can see your Population variable under the Marks window, formatted as a Measure (SUM). You can also see the legend for the plot being built automatically on the right side, with the sizes of various circles given by population. Change the population variable from a Measure to an Attribute.

The scatter plot is still very busy. Let’s restrict our graphic to census block groups with populations over 3,000 residents. Drag the Population variable over to the Filters window. It will open up the dialogue box below automatically, prompting you to choose the format first. Choose Attribute again and hit Next.

Now set the lower bound of the population filter to 3,000 and hit OK.

As you can see, the plot is less crowded now. Our legend automatically updated to become specific to our new range. Now let’s try adding a color scale to our plot. Drag the variable “Race White Perc” over to the Color card in Marks and change it to an Attribute again.

Another legend box was added for the percentage of the population responding with their race as white. You can start to see that a cluster of observations in the center-left of the chart is darker than many others, indicating that those census block groups have a higher proportion of white residents. But we still have too much clustering in the bottom left to know if this is a significant cluster.

Drag the variable, Edu Master or Prof Degr Perc, to the filter window. You’ll be prompted with another set of dialogue boxes; choose Attribute again and set the minimum to 0.04. The observations with a percentage of residents with master degrees below 0.04 will be filtered from the graph and the axis range will adjust automatically.

You can also add a filter by right clicking over in the legend area and selecting Filters. Choose the first variable, “Edu Master or Prof Degr Perc” to include a live filter bar.

You can interactively adjust the filters for your plot using the legend cards on the right side. Try adding the “Pop.Estimate” filter to your legend as well and play around with different values on the slider bars. Once you publish your plot you can even keep the interactivity of these filters, allowing your audience to adjust the filter values themselves.

As a final element for our scatterplot let’s add a line of best fit. Click on the “Analysis” drop down menu from the top bar and click on “Trend Lines”, then select “Show Trend Lines”. This will be the easiest regression you have ever run!

After you have added a trend line, you can hover over the line in the graph window to view the regression equation, including the coefficient, intercept value, R-Squared, and p-value!

If you right click on the trend line itself you can choose to “Describe Trend Model…”, which will pull up the following window with some familiar output.

Now you have the number of observations, degrees of freedom, standard errors, t-statistics, and p-values! Makes you wonder why we ever do regressions in Stata, right? Unfortunately, while Tableau has surprisingly powerful analysis tools, it can’t match Stata in its ability to build and test regression models.

Connecting Data

Making plots based on numeric data is fun and helpful, but the real magic behind Tableau is its ability to connect different datasets and build visualizations drawing from each of them. Let’s connect our demographic data with the spatial data containing census block group boundaries. Return to the Data Source tab on the bottom left and then click “Add” next to the Connections list.

Click the option for “Spatial file” and browse to the “CBG_2017” sub-folder. Inside you should find the shapefile “cb_2017_36_bg_500k.shp”. You’ll notice that the only file that appears is the shapefile, not any of the other files in the folder, which are all support files for the .shp.

Immediately after you hit “Open” Tableau will prompt you to select variables to join the datasets on. Tableau automatically wants to connect both datasets according to a unique ID variable, much like the “merge” command in Stata. Our unique ID in the ACS .csv file is the “Id” variable, while the connected ID in the shapefile is the “Affgeoid” variable. Tableau will ask for both inside the Join menu, which is represented as a Venn-Diagram. We will use an “Inner Join”, which connects observations 1:1 and only shows observations for which there are matches in both datasets. For more on different types of Joins, see Tableau’s guide here.

You can now see the variables from both data files in the browser, with variables color coded according to which data set they come from. Shapefiles, while they contain spatial data, also hold numeric and string data just like a .csv file for each observation. This allows us to link them easily with other data types and store new information in them connected to geometry.

Creating Map Visualizations

Let’s make a new worksheet for our map visualization. Use the “New worksheet” button at the bottom of Tableau to open Sheet 2. You’ll see on the left our variables now include both the shapefile and ACS data. To start with your basemap, drag the “Geometry” Measure from the shapefile onto your plot window. Tableau automatically formats shapefile geometry as its own variable, so you don’t need to worry about longitude and latitude coordinates for a map.

New York State is pulled up on your plot view, with the longitude and latitude automatically split into the Rows and Columns spaces up top and the Geometry variable in the Marks window, formatted as a Measure (COLLECT). You’ll notice this forces the entire state to act as a single observation. To specify particular census block groups, drag the variable “Geometry” from the ACS data Dimensions onto the Detail card under the Marks window.

Now each census block group is its own observation, with a name associated to it. If you hover over the map it will pull up the name--if you click it will filter the map automatically to that census block group.

Let’s add more details to the map to match our plot from before. Let’s start with the “Race White Perc” variable as a Color. Don’t forget to change it to an Attribute.

Now when you hover over the census block groups it will show the percentage of the population responding as white. You can add a filter bar in the legend by right clicking like we did before. Then we can add our “Pop.Estimate” variable and filter to census block groups with populations greater than 3,000 again.

What do you think of our sample now? Does it seem appropriate to filter only on population? We have a collection of census block groups from around the state with presumably very different populations and contexts. Our simple plot from before would not have told us this, but with a shapefile we can observe the spatial relationship between observations.

If we want to zoom or pan around the map to examine New York City in particular we can use the drop down menu, similar to Google Maps. You can change whether clicking on the map pans (shortcut f) or selects (shortcut a) using the menu in the play-button triangle.

Let’s relax our population filter to 1,000 and then select the area around New York City.

Right click on your selection and press “Keep Only”. Tableau now associates a sub-sample of your data with the Sheet 2 map and will work only with those observations.

Let’s make our map a little more interpretable by removing the black borders around census block groups. Click on the Color card and then under the Border drop down select “None”.

Note: If you want to bring back all the observations, rather than filtering by the geography around NYC, you can add a filter to the legend on the right side and select “(All)”.

Zooming in more, we can start to observe that Queens and the Bronx seem to have lower proportions of white residents than other areas like Manhattan.

It’s more difficult to represent multiple variables simultaneously via maps, but the benefits of spatial geometry more than compensate. You can keep adding filters or color attributes and produce maps of the same area to compare multiple variables, then display them in a single window using a Tableau Dashboard.

Creating Dashboards

Tableau’s biggest advantage over similar visualization programs, other than its connections, is its ability to make Dashboards of multiple visualizations. At the bottom of Tableau, select the “New Dashboard” button to create Dashboard 1.

Creating Dashboards is incredibly simple. Just drag and drop the Sheets you made before into the window that says “Drop sheets here”. Try Sheet 2 first--this may take a moment.

You can interact with Sheets in the Dashboard as you would in the Sheet tab, adjusting filters or map views as necessary. You can also drag in other sheets to view side-by-side, either vertically or horizontally stacked with the first. Drag Sheet 1 above Sheet 2, making sure that the gray highlighted area corresponds to the top half of the Dashboard.

You can see on the Dashboard bar on the left that you can choose to optimize the size of the Dashboard for a desktop browser or mobile phone, depending on your audience.

You can also create a Dashboard that links the two Sheets, so that filtering one affects the other or selecting an observation in the map can highlight the relevant circle in the plot.

You can likewise export Dashboards as image files or publish them to a Story, which allows you to order Dashboards or Sheets into Slides similarly to a PowerPoint presentation. The most interesting way to export all of these, however, is to publish them to the Tableau Server.

Tableau Server & Hosting Projects

In order to publish your Worksheet to the Tableau Server you will first need to link your account to a Tableau Public account. You may need to create a new account there under the same email address you used to download Tableau Desktop. After you’ve done that, the first step to hosting on Tableau Public is to extract your data, changing them from the “Live” option to the “Extract” option in the Data Source tab. An important note: if you publish to Tableau Public your data will be hosted publicly. Make sure to cite your sources properly and ensure that you have permission to publish the data before you proceed!

QGIS Environment

QGIS has become the leading standard in open source spatial data software. Its primary functions are importing, manipulating, and visualizing geographic data, though it also has useful tools for connecting non-spatial data and performing simple analysis. It has a standard, photoshop-style user interface based on layers of data displayed in a visualization window, with a host of tools and settings in the bars at the top.

There are many options for customizing the QGIS environment, but the main addition that I prefer is the “Manage Layers Toolbar”. You can activate it in the View -> Toolbars dropdown.

Spatial Vector Data

The primary type of spatial data which you can work with is vector files. These are often traditional shapefiles, a designation created by Esri (the folks behind ArcGIS), but can also include KML and KMZ filetypes. Fundamentally they are all collections of coordinates organized into shapes (points, lines, and polygons). These shapes are often coded with unique identifiers and connected to typical numeric, text, or boolean data variables. Shapefiles are almost always associated with a database support file, which QGIS can import with the geometry as an associated data table, much like a CSV file.

First, let’s bring in a shapefile of all census block groups in the state of New York using the “Add Vector Layer” button in our Manage Layers Toolbar.

This brings up the Data Source Manager opened to the Vector tab. You can browse for a vector dataset using the button with an ellipse “...” next to the filepath bar.

After browsing to the “CBG_2017” folder, select the shapefile “cb_2017_36_bg_500k.shp”. Note that shapefiles use the .shp file format, while the various support files associated with them can often include similar formats like .shx. Ensure that you select the type “SHP File”.

After you successfully import the shapefile you should see the state of New York, divided by census block groups, as below. The color is selected randomly so don’t worry if it isn’t yellow.

If you want to see the data variables associated with the shapefile, you can open the browse window at the top right, clicking the button “Open Attribute Table”, shortcut F6.

Basemaps using QuickMapServices Plugin

Most maps include some form of base layer, usually a map of political boundaries or satellite imagery. These can be tiresome to find and download online for your specific map extent. Programmers, being lazy, have thus developed packages to automatically download and import a base layer. These user created packages are maintained by volunteers and as such can be out of date for newer versions of QGIS, but most “stable builds” have a full suite of packages for baselayers. If you are using version 3.4 or below you should have access to the QuickMapServices plugin, which hosts maps made by the Open Street Map (OSM) project, one of the largest open source mapping projects in the world.
